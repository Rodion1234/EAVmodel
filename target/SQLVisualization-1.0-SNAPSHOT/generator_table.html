Entity–attribute–value model (EAV) is a very flexible data model used in various open source projects. This data model has many advantages and disadvantages so it is not suitable for wide application. But sometimes it is highly recommended to use EAV in design and development process. Cases are decribed, in which it is preferable to use EAV data model. EAV is completely compared with the standard relational data model and some examples of successfully integration EAV in ready-to-use open source solutions are given.
The main advantages of the EAV data model are:
  • Flexible and universal data structure (you can change the number of properties without changing the fields in the table);
  • Relative simplicity when adding or changing product characteristics;
  • When properly implemented, data redundancy is almost absent.

The main cons of EAV:
  • To insert data is usually used several queries;
  • Not very fast data sampling in the case of implementation with a large number of tables and high server load with large amounts of data due to the inability to use standard indexing methods;
  • Sometimes there are difficulties in ensuring data integrity.
